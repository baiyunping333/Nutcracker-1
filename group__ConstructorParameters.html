Detailed Description

C++ constructors can only take the name of the class itself, so it is impossible to distinguish different ways in which a class can be constructed by using constructors with different names. Fortunately, C++ allows one to create several methods in a class that share the same name but that have different types and numbers of arguments. Thus, several "wrapper" classes have been created that are nothing more than a simple wrapper around an argument in order to explicitly specify a particular constructor. Associated with each such class is also a convenience function for wrapping a value in this class, so that for example one can construct a tensor B that as a copy of a tensor A via code like the following:

OperatorTensor B(copyFrom(A));

where copyFrom(A) constructs a value of type CopyFrom<OperatorSite> and passes it to the constructor, which causes C++ (using the argument-dependent lookup rules) to select the constructor that we want --- the one that will make a copy of the data in A.

A convenience macro for constructing constructor parameter wrappers.

For example, the CopyFrom class is defined using the following line of code:

DEFINE_TEMPLATIZED_PARAMETER(CopyFrom,copyFrom)

(Note that the two parameters are essentially the same thing, but using different capitalization.)
