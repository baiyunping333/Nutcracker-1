#define BOOST_LOCAL_CONFIG_FUNCTION_ARITY_MAX   5

Maximum number of function parameters supported for the local functions.

If programmers leave this macro undefined, its default value is 5.

This only refers to the number of local function parameters and not to the number of bound variables in scope (the limit on the number of bound variables is instead the maximum size allowed for a Boost.Preprocessor sequences).

Increasing this number will increase compilation time.

#define BOOST_LOCAL_CONFIG_THIS_PARAM_NAME   this_

The name of the special symbol used to access the bound object this.

If programmers leave this macro undefined, the default symbol used is this_.

Programmers should not define this macro unless it is absolutely necessary (e.g., to avoid name clashes with another library which cannot be changed). Changing the symbol this_ effectively changes the public API of this library.
