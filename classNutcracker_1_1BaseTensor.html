Detailed Description

The base class of all tensors.

All tensors in Nutcracker share in common the trait that they are either invalid or they contain a pointer to a chunk of contiguous complex<double> data of known size. This base class implements some natural functionality that follows from this trait (such as the ability to query the size of the data, or to get a pointer to it) that is shared amongst all of the tensors classes.

A BaseTensor object holds sole ownership over its data and will automatically free it when the destructor is called. Two objects cannot share data, but you can move data from one object to another. For example, the code 

a = boost::move(b);

moves the data from b to a and then invalidates b. Of course, this "move" only involves moving the data pointer and size fields and not the data itself, so it is a very cheap operation. The boost::move function (part of the Boost.Move library) does some magic that you don't have to know about, so you can just think about it as being required to confirm that you know that you are moving the data out of b and so b will be invalid after this statement.

In many cases a function will return a value that is an instance of BaseTensor. The Boost.Move library does magic behind the scenes so that if f is a function returning an OperatorTensor then you can write 

OperatorTensor a(f(...));

to indicate that the result of f should be moved into a.

Note that the interface of BaseTensor has been designed so that copies will never be made unless you explicitly ask for them, so if you fail to use the correct syntax to move the data between objects then you will get a compiler error rather than having a copy be silently made.
