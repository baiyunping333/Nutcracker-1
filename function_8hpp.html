This macro starts the declaration of a local function.

This macro must be used within a declarative context, it must be followed by the local function body code { ... } and then by either the {BOOST_LOCAL_FUNCTION_END} or the {BOOST_LOCAL_FUNCTION_END_RENAME} macro (see the {Tutorial} and {Advanced} sections): 

  { // Some declarative context.
      ...

      BOOST_LOCAL_FUNCTION(
      parenthesized_signature
      ) exception_specifications_optional {
          ... // Block body.
      } BOOST_LOCAL_FUNCTION_END(local_function_name)

      ...
  }

Within templates, the special macro {BOOST_LOCAL_FUNCTION_TPL} must be used instead of {BOOST_LOCAL_FUNCTION}.

{parenthesized_signature, A Boost.Preprocessor sequence that uses the parenthesized syntax to specify the local function signature and the variables in scope to bind (see the {Grammar} section). }

As usual, exception specifications can be optionally programmed before the body code block (see the {Advanced} section).

#define BOOST_LOCAL_FUNCTION_END_RENAME (   new_local_function_name )    BOOST_LOCAL_AUX_FUNCTION_END_RENAME(new_local_function_name)

This macro ends the definition of a local function while changing the function name.

This macro must be used instead of the {BOOST_LOCAL_FUNCTION_END} macro and it must follow the local function body code { ... } as shown in the {BOOST_LOCAL_FUNCTION} documentation.

{new_local_function_name, This is the new local function name (see the {Advanced} section). }

The original local function name (as specified by the parenthesized signature passed to {BOOST_LOCAL_FUNCTION}) must still be used within the local function body (e.g., for the local function to recursively call itself). However, within the enclosing scope the new local function as passed to {BOOST_LOCAL_FUNCTION_END_RENAME} must be used (e.g., to invoke the local function).

Renamed local functions cannot be directly passed as template parameters (e.g., to STL algorithms). However, it is possible to workaround this limitation by getting a reference to the renamed local function using boost::local::{boost/local/,function_ref} and then by passing the local function reference as template parameter.
