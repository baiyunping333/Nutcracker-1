00014 // INFIX is used to separate symbols concatenamed together. Some of these
00015 // symbols are user defined so they can be anything. Because they can contain
00016 // underscore `_` and/or start with capital letters, it is NOT safe for the
00017 // INFIX to be the underscore `_` character as that could lead to library
00018 // defined symbols containing double underscores `__` which are reserved.
00019 // C++ standard: (1) "each name that contains a double underscore (_ _) or 
00020 // begins with an underscore followed by an uppercase letter is reserved to 
00021 // the implementation" and (2) "each name that begins with an underscore is 
00022 // reserved to the implementation for use as a name in the global namespace".
00023 #define BOOST_LOCAL_AUX_SYMBOL_INFIX X // `X` used as separator.
